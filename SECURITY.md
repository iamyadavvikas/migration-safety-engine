# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

Migration Safety Engine handles database schema changes. If you discover a security vulnerability, please report it privately.

**Do not** report security vulnerabilities through public GitHub issues.

Instead, report via email to **vikasyadav.dev@gmail.com** with subject line starting with `[MSE-SEC]`.

You should receive a response within 48 hours. If you don't, please follow up.

## Scope

- Unauthorized access to migration state
- Privilege escalation via migration plans
- Injection attacks in plan YAML/JSON parsing
- Drift detection bypass

## Safe Harbor

We consider security research and vulnerability disclosure to be protected under coordinated disclosure. We will not take legal action against researchers who:

- Follow this policy
- Make a good-faith effort to avoid privacy violations
- Do not destroy data
- Report privately
